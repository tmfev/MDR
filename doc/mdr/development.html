Development

If you want to create a WAR-file from the source code, you'll find the following information helpful.

Prerequisites

Some tools need to be present on your development machine:

Tool Location
Java http://www.oracle.com/technetwork/java
Apache Ant http://ant.apache.org
Git http://git-scm.com

Getting the Source

The source code lives in a Git repository and you will need a local copy of the repository. Due to the nature of Git, you can clone any repository that holds the MDR project.The project is hosted on Github as github.com:tmfev/MDR.git, open the project there for further information.

A typical command to clone the repository looks like this: 

	git clone git@github.com:tmfev/MDR.git

The previous command will check out the HEAD revision. If you want to build a specific version, you need to check it out first: 

	git checkout 0.8.37

Building the WAR

There is a single WAR file that constitutes the runtime components which is called ItemDesigner.war. To create this WAR file, type 

	ant war

Once you have the WAR file, it's time to deploy.

API Documentation

The Java API Documentation has been created using JavaDoc. It is available after running ant doc.

Along with the API documentation comes statistics about the quality of the documentation using DocCheck
